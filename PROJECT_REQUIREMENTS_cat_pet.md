# 网页版电子宠物小猫项目需求文档

## 1. 项目名称

**小猫成长记：儿童互动电子宠物网页应用**

## 2. 项目目标

开发一个面向小朋友使用的网页版电子宠物小猫应用。小朋友可以通过完成任务获得积分和成长值，让小猫不断成长、升级、换装，并使用积分兑换食物、玩具、服装和场景装饰。

项目重点：

- 界面简洁可爱
- 适合儿童操作
- 使用图标 + 简短文字
- 有完整成长系统
- 有积分加减系统
- 有积分兑换系统
- 有小猫状态反馈
- 可以在浏览器中直接运行

---

## 3. 技术要求

### 推荐技术栈

请使用以下技术实现：

- React
- TypeScript
- Vite
- CSS / CSS Modules
- localStorage 本地存储数据

第一版不需要后端，所有数据先保存在浏览器 localStorage 中。

### 页面适配

需要支持：

- 桌面端浏览器
- 平板浏览器
- 手机浏览器

界面需要响应式布局，按钮要大，方便儿童点击。

---

## 4. 产品定位

这是一个儿童互动成长类网页应用。

小朋友每天进入网页后，可以看到自己的小猫。通过完成任务获得积分和成长值。积分可以用来兑换小猫的食物、玩具、衣服和房间装饰。成长值可以让小猫升级，从小奶猫成长为明星小猫。

整体风格应当是：

- 可爱
- 圆润
- 温暖
- 儿童友好
- 简洁明亮
- 图标清晰
- 文字简短

---

## 5. 核心功能

### 5.1 小猫主页

主页是应用的核心页面。

主页需要显示：

- 小猫名字
- 当前等级
- 当前积分
- 当前成长值进度条
- 小猫形象
- 小猫状态
- 快捷操作按钮

快捷按钮包括：

- 喂一喂
- 洗一洗
- 玩一玩
- 做任务
- 去商店
- 换装

小猫形象可以先用 CSS、SVG、emoji 或简单插画占位实现，后期可以替换成正式图片素材。

---

### 5.2 小猫状态系统

小猫有以下状态值：

| 状态 | 字段 | 满分 | 说明 |
|---|---|---:|---|
| 饱腹值 | hunger | 100 | 小猫是否饿了 |
| 心情值 | mood | 100 | 小猫是否开心 |
| 清洁值 | clean | 100 | 小猫是否干净 |
| 体力值 | energy | 100 | 小猫是否有精神 |

状态显示方式：

- 使用图标 + 数字 / 进度条
- 图标要适合儿童理解
- 状态低时显示温和提醒

状态图标建议：

| 状态 | 图标建议 |
|---|---|
| 饱腹值 | 小鱼干 |
| 心情值 | 爱心 |
| 清洁值 | 泡泡 |
| 体力值 | 闪电 |

状态变化规则：

- 喂食：饱腹值 +20
- 洗澡：清洁值 +20
- 玩耍：心情值 +15，体力值 -5
- 完成任务：心情值 +5，成长值增加
- 任务失败或放弃：心情值 -5

状态最低为 0，最高为 100。

---

### 5.3 成长系统

小猫通过成长值升级。

成长阶段如下：

| 阶段 | 等级范围 | 阶段名称 | 说明 |
|---|---|---|---|
| 1 | Lv.1 - Lv.3 | 小奶猫 | 初始形态，圆润可爱 |
| 2 | Lv.4 - Lv.7 | 活力小猫 | 解锁跳跃、玩耍动作 |
| 3 | Lv.8 - Lv.12 | 聪明小猫 | 解锁学习类装扮 |
| 4 | Lv.13 - Lv.18 | 冒险小猫 | 解锁更多场景 |
| 5 | Lv.19+ | 明星小猫 | 解锁高级服饰和称号 |

升级规则：

```ts
const levelRules = [
  { level: 1, needGrowth: 0 },
  { level: 2, needGrowth: 50 },
  { level: 3, needGrowth: 120 },
  { level: 4, needGrowth: 220 },
  { level: 5, needGrowth: 350 },
  { level: 6, needGrowth: 520 },
  { level: 7, needGrowth: 720 },
  { level: 8, needGrowth: 950 }
]
```

当成长值达到升级条件时：

- 自动升级
- 显示“升级啦！”
- 播放简单动画
- 小猫心情值 +10
- 可以解锁新商品或新装扮

---

### 5.4 任务系统

任务是小朋友获得积分和成长值的主要方式。

任务页面需要展示今日任务列表。

任务卡片包含：

- 任务图标
- 任务名称
- 奖励积分
- 奖励成长值
- 任务状态
- 完成按钮

任务状态包括：

- 未完成
- 已完成
- 已领取

第一版可以不做家长审核，点击“我完成啦”后直接获得奖励。

默认任务示例：

| 任务 | 图标 | 积分 | 成长值 |
|---|---|---:|---:|
| 阅读 10 分钟 | 📖 | 10 | 5 |
| 自己整理书包 | 🎒 | 8 | 5 |
| 按时刷牙 | 🪥 | 5 | 3 |
| 跳绳 100 下 | 🏃 | 10 | 6 |
| 主动说谢谢 | 🙌 | 5 | 3 |
| 画一幅画 | 🎨 | 12 | 8 |

完成任务后：

- 积分增加
- 成长值增加
- 小猫心情值增加
- 显示鼓励提示

鼓励文案示例：

- 太棒啦！
- 小猫很开心！
- 你帮助小猫成长啦！
- 今天也很努力！
- 再完成一个任务就能升级哦！

---

### 5.5 积分系统

积分用于兑换商品。

积分增加来源：

| 行为 | 积分 |
|---|---:|
| 完成简单任务 | +5 |
| 完成普通任务 | +10 |
| 完成困难任务 | +15 |
| 每日首次登录 | +5 |
| 小猫升级 | +20 |

积分减少来源：

| 行为 | 积分 |
|---|---:|
| 兑换食物 | -10 到 -30 |
| 兑换玩具 | -20 到 -50 |
| 兑换服装 | -50 到 -150 |
| 兑换背景 | -80 到 -200 |

扣分文案要温和，不要有惩罚感。

示例：

- 小猫有点饿啦，下次记得照顾它哦。
- 积分用来兑换了新礼物！
- 没关系，下次继续努力！

---

### 5.6 商店兑换系统

商店页面需要展示可兑换商品。

商品分类：

- 食物
- 玩具
- 服装
- 背景
- 特效

商品卡片包含：

- 商品图标
- 商品名称
- 商品价格
- 商品类型
- 兑换按钮
- 是否已拥有

默认商品示例：

| 商品 | 类型 | 图标 | 价格 | 效果 |
|---|---|---|---:|---|
| 小鱼干 | 食物 | 🐟 | 20 | 饱腹值 +20 |
| 牛奶 | 食物 | 🥛 | 25 | 饱腹值 +25 |
| 毛线球 | 玩具 | 🧶 | 30 | 心情值 +15 |
| 小帽子 | 服装 | 🎩 | 60 | 解锁装扮 |
| 小围巾 | 服装 | 🧣 | 80 | 解锁装扮 |
| 草地背景 | 背景 | 🌱 | 100 | 解锁背景 |
| 星空背景 | 背景 | 🌟 | 150 | 解锁背景 |

兑换逻辑：

- 积分足够时可以兑换
- 积分不足时按钮置灰
- 已拥有商品显示“已拥有”
- 食物类兑换后可以立即使用
- 服装和背景兑换后加入背包
- 兑换成功后显示弹窗反馈

---

### 5.7 背包 / 换装系统

背包页面展示已经拥有的物品。

物品类型：

- 食物
- 玩具
- 服装
- 背景

功能：

- 使用食物
- 使用玩具
- 穿戴服装
- 切换背景

第一版可以实现简单换装逻辑：

- 当前帽子
- 当前围巾
- 当前背景

小猫主页需要根据当前装扮显示对应文字或图标效果。

## 15. 验收标准

项目完成后需要满足以下条件：

1. 打开网页可以看到小猫主页。
2. 首页能显示小猫名字、等级、积分、成长进度和状态。
3. 点击“喂一喂”后，饱腹值增加。
4. 点击“洗一洗”后，清洁值增加。
5. 点击“玩一玩”后，心情值增加。
6. 进入任务页面，可以完成任务。
7. 完成任务后，积分和成长值增加。
8. 成长值达到条件后，小猫自动升级。
9. 进入商店页面，可以兑换商品。
10. 积分不足时不能兑换。
11. 兑换成功后，积分减少。
12. 已兑换商品可以在背包中看到。
13. 刷新页面后数据仍然存在。
14. 页面适合儿童使用，按钮大，文字清楚。
15. 手机端和电脑端都能正常显示。
